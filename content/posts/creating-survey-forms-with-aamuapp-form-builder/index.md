<p xmlns="http://www.w3.org/1999/xhtml">Aamu.app Forms let you build survey forms, feedback forms, sign-up forms, internal request forms, and small data-collection workflows without wiring together a separate form tool and a separate spreadsheet. The form is public when you publish it, and the responses are stored in an Aamu database table.</p><p xmlns="http://www.w3.org/1999/xhtml">That database connection is the important part. A form is not just a page that sends you an email. Each submission becomes structured data that you can review, filter, export through your workflow, automate, or use from the Aamu API.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Create a new form</h2><p xmlns="http://www.w3.org/1999/xhtml">Open the Forms section in the project where the survey should live, then create a new form. Forms are project-scoped, so choose the project deliberately: the project controls where the form appears, which team members work with it, and where the connected response database belongs.</p><p xmlns="http://www.w3.org/1999/xhtml">A new form starts as a draft. Give it a clear title and, if useful, a short description that tells respondents what the form is for. For a survey, this is usually enough:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>what you are asking about,</p></li><li><p>how long it will take, and</p></li><li><p>what happens after the respondent submits it.</p></li></ul><h2 xmlns="http://www.w3.org/1999/xhtml">Add survey fields</h2><p xmlns="http://www.w3.org/1999/xhtml">Use <strong>Add new field</strong> to build the form. The current field types cover the common survey cases:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p><strong>Text answer</strong> for short free-text answers,</p></li><li><p><strong>Paragraph</strong> for longer open answers,</p></li><li><p><strong>Choice</strong> for radio buttons, dropdowns, or checkboxes,</p></li><li><p><strong>Date/time</strong> for dates, times, or date-time values, and</p></li><li><p><strong>File upload</strong> for attaching files to the response.</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">Each field can have a title, description, placeholder text where relevant, and a required toggle. Choice fields can be one-choice radio buttons, one-choice dropdowns, or multiple-choice checkboxes.</p><p xmlns="http://www.w3.org/1999/xhtml">For surveys, avoid making every field required. Required fields are useful for must-have data such as email address or consent, but too many required fields make people abandon the form.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Think in database columns</h2><p xmlns="http://www.w3.org/1999/xhtml">When you publish a form, Aamu connects the form to a database table. The form fields become structured columns, and each submission creates a row.</p><p xmlns="http://www.w3.org/1999/xhtml">This has two practical consequences:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>After a field has been published into the database, its type is part of the response structure.</p></li><li><p>If you change a published form, publish the new elements so the database table and public form stay in sync.</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">This is why the editor distinguishes draft changes from published form structure. You can keep editing, but the public form and the response database need an explicit publish step when the structure changes.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Configure form settings</h2><p xmlns="http://www.w3.org/1999/xhtml">Open the form Settings tab before publishing. The useful settings are:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p><strong>Short url</strong> for a cleaner public link,</p></li><li><p><strong>Thank-you page</strong> for redirecting successful submissions,</p></li><li><p><strong>Error page</strong> for redirecting failed submissions,</p></li><li><p><strong>Submit button text</strong> for matching the form's purpose, and</p></li><li><p><strong>Custom CSS</strong> for adjusting the public form's appearance.</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">If you do not set custom thank-you or error pages, Aamu uses default pages. For public surveys, a custom thank-you page is often worth adding because it can tell the respondent what happens next.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Publish and share the form</h2><p xmlns="http://www.w3.org/1999/xhtml">Publish the form when it is ready. After publishing, Aamu shows a <strong>View form</strong> link. The public URL uses this shape:</p><pre xmlns="http://www.w3.org/1999/xhtml"><code class="language-plaintext">https://YOUR_AAMU_SUBDOMAIN.aamu.app/shared/form/FORM_ID_OR_SHORT_URL</code></pre><p xmlns="http://www.w3.org/1999/xhtml">Use the generated link from Aamu. If you configured a short url, the public link can use that instead of the raw form id.</p><p xmlns="http://www.w3.org/1999/xhtml">Before sending the link to real respondents, submit the form once yourself. Check the thank-you page, required fields, choice labels, file upload fields if any, and the resulting response row.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Review responses</h2><p xmlns="http://www.w3.org/1999/xhtml">Once the form is public, the form header includes a Responses tab. Aamu shows both a summary view and the underlying database connection.</p><p xmlns="http://www.w3.org/1999/xhtml">The summary view is useful for quick survey reading: choice fields can be charted, and text fields can be reviewed as collected answers. The details view shows the actual database rows. Aamu also links directly to the database table, where you can work with the responses like any other database data.</p><p xmlns="http://www.w3.org/1999/xhtml">This is helpful when a survey turns into work. For example:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>a feedback form can create follow-up tasks,</p></li><li><p>a sign-up form can become a customer or lead list,</p></li><li><p>a bug-report form can be triaged in a database view, and</p></li><li><p>a file-upload form can keep attachments connected to the response row.</p></li></ul><h2 xmlns="http://www.w3.org/1999/xhtml">Automate after responses arrive</h2><p xmlns="http://www.w3.org/1999/xhtml">Because responses are stored in a database table, you can use database automations for follow-up work. For example, a response can lead to a task, a notification, a status change, or another internal workflow.</p><p xmlns="http://www.w3.org/1999/xhtml">The form itself collects the data. The database and automations are where the response becomes part of the team's work.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Using the Forms API</h2><p xmlns="http://www.w3.org/1999/xhtml">For normal public surveys, use the published form link. A browser visitor does not need an API key to submit the public form.</p><p xmlns="http://www.w3.org/1999/xhtml">For integrations, Aamu also has an authenticated Forms API. API keys can list forms, fetch a form definition, and submit form responses with Forms scope:</p><pre xmlns="http://www.w3.org/1999/xhtml"><code class="language-plaintext">GET  /api/v1/forms/
GET  /api/v1/forms/{id}
POST /api/v1/forms/{id}/submissions</code></pre><p xmlns="http://www.w3.org/1999/xhtml">Use the API when another system needs to submit structured responses into Aamu. Use the public form when a person is filling in a survey in the browser.</p><h2 xmlns="http://www.w3.org/1999/xhtml">A simple survey checklist</h2><ol xmlns="http://www.w3.org/1999/xhtml"><li><p>Create the form in the right project.</p></li><li><p>Add a short title and description.</p></li><li><p>Add fields with clear labels.</p></li><li><p>Mark only the necessary fields as required.</p></li><li><p>Configure the short url and thank-you page.</p></li><li><p>Publish the form.</p></li><li><p>Open the public link and submit a test response.</p></li><li><p>Check the Responses tab and the database row.</p></li><li><p>Add automations if responses should trigger follow-up work.</p></li></ol><p xmlns="http://www.w3.org/1999/xhtml">That is the main advantage of building surveys inside Aamu.app: the form, the response data, and the follow-up work can live in the same project instead of being split across a form builder, a spreadsheet, and a task tool.</p>

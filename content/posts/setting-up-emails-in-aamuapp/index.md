<p xmlns="http://www.w3.org/1999/xhtml">Let’s tackle a bigger subject — emails. Due to the nature of the game, it involves some pretty techy stuff, domains and other scary things. This is the same for Helpdesk, which shares those domain settings with Emails.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Email domain</h2><p xmlns="http://www.w3.org/1999/xhtml">Emails work so that one project can hold one <em>Email domain</em>. This project can hold one email account for each project user. Those email accounts are personal — others won’t be able to see your email account’s emails or contacts.</p><p xmlns="http://www.w3.org/1999/xhtml">Email domains are created in <em>Team settings</em>. First you need to add an email domain. Let’s do that now.</p><p xmlns="http://www.w3.org/1999/xhtml">Go to Profile icon <span data-node="emoji" data-emoji="➡️" data-annotation="right arrow" style="user-select: text; font-family: &quot;Twemoji Mozilla&quot;, &quot;Apple Color Emoji&quot;, &quot;Segoe UI Emoji&quot;, &quot;Segoe UI Symbol&quot;, &quot;Noto Color Emoji&quot;, &quot;EmojiOne Color&quot;, &quot;Android Emoji&quot;, sans-serif;"><img alt="right arrow" style="display: none; width: 1em; height: 1em;" ><span role="img" aria-label="right arrow" style="display: inline-block;">➡️</span></span> Team <span data-node="emoji" data-emoji="➡️" data-annotation="right arrow" style="user-select: text; font-family: &quot;Twemoji Mozilla&quot;, &quot;Apple Color Emoji&quot;, &quot;Segoe UI Emoji&quot;, &quot;Segoe UI Symbol&quot;, &quot;Noto Color Emoji&quot;, &quot;EmojiOne Color&quot;, &quot;Android Emoji&quot;, sans-serif;"><img alt="right arrow" style="display: none; width: 1em; height: 1em;" ><span role="img" aria-label="right arrow" style="display: inline-block;">➡️</span></span> Email domains. To add a new domain, click the plus button:</p><img xmlns="http://www.w3.org/1999/xhtml" src="bb63207d60b97580_image.png" style="width: auto; --upload-progress: 0%;" id="3bf970f6-16ae-497e-98ff-e1bc3defd5f0"><p xmlns="http://www.w3.org/1999/xhtml">And proceed to add the domain. Note that you can choose between Aamu.app’s email (IMAP) server and a third party IMAP server. So, that’s some extra flexibility. </p><p xmlns="http://www.w3.org/1999/xhtml">Let’s use Aamu.app’s IMAP server.</p><img xmlns="http://www.w3.org/1999/xhtml" src="92ac87bf00243cea_image.png" style="width: auto; --upload-progress: 0%;" id="88501f9c-45a8-4103-b85a-731b4cccf994"><p xmlns="http://www.w3.org/1999/xhtml">After we have added it, we get to the screen where the domain will be verified through the DNS data. For this you need to get to the site that holds the DNS data for your domain, for example Cloudflare. The data that you should enter into your domain’s DNS data looks like this:</p><img xmlns="http://www.w3.org/1999/xhtml" src="f7b573b9f9ce152d_image.png" style="width: auto; --upload-progress: 0%;" id="3fc0b2df-f2c6-4589-86d0-1a8237d82f6d"><p xmlns="http://www.w3.org/1999/xhtml">There are four fields to set. So just set them, copy-paste from the screen to the site that holds your site’s DNS data. After clicking <em>Verify</em> you should hopefully see this:</p><img xmlns="http://www.w3.org/1999/xhtml" src="c6a1dee8e74692ca_image.png" style="width: auto; --upload-progress: 0%;" id="1e41ef99-d1d7-4de6-9ec0-87d1fb73949a"><p xmlns="http://www.w3.org/1999/xhtml">If the verification didn’t succeed, just try again until it does.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Attaching the domain to the project</h2><p xmlns="http://www.w3.org/1999/xhtml">After the hard part, DNS verification is done, there is still one more step until we get to create our mailboxes. </p><img xmlns="http://www.w3.org/1999/xhtml" src="40b777ed5a6233c4_image.png" style="width: auto; --upload-progress: 0%;" id="d55ae359-7f2a-4d8a-8fa5-bc154b7eb153"><p xmlns="http://www.w3.org/1999/xhtml">This should be simple enough.

<p xmlns="http://www.w3.org/1999/xhtml">Aamu.app has keyboard shortcuts for the actions you repeat all day: moving between sections, creating new items, navigating lists, selecting items, closing tasks or tickets, and opening comment panels.</p><p xmlns="http://www.w3.org/1999/xhtml">The shortcuts are intentionally contextual. A key usually activates the visible button or item for the current view. That keeps the same habit useful across Tasks, Docs, Emails, Helpdesk, Meetings, Forms, Databases, Files, Designs, Chat, and Posts.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Shortcuts are visible in the UI</h2><p xmlns="http://www.w3.org/1999/xhtml">Many Aamu buttons show their shortcut in the tooltip. For example, a create button may say <code>New doc [c]</code>, <code>Compose new email [c]</code>, or <code>New meeting [c]</code>.</p><p xmlns="http://www.w3.org/1999/xhtml">This is the best way to learn shortcuts: hover the control you already use and notice the key in brackets.</p><h2 xmlns="http://www.w3.org/1999/xhtml">When shortcuts do not run</h2><p xmlns="http://www.w3.org/1999/xhtml">Aamu avoids running global shortcuts while you are typing into an input, textarea, editor, or checkbox. That means ordinary writing should not accidentally navigate away or close an item.</p><p xmlns="http://www.w3.org/1999/xhtml">In practice: if your cursor is inside a text field, the text field gets the key. If you are on the page itself, Aamu can use the key as a shortcut.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Navigation shortcuts</h2><p xmlns="http://www.w3.org/1999/xhtml">The sidebar has number shortcuts for the main sections. The exact sections depend on which apps your team has enabled, but the common mapping is:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p><code>1</code> Home</p></li><li><p><code>2</code> Posts or AI, depending on the enabled sidebar layout</p></li><li><p><code>3</code> Chats</p></li><li><p><code>4</code> Tasks</p></li><li><p><code>5</code> Helpdesk</p></li><li><p><code>6</code> Emails</p></li><li><p><code>7</code> Designs</p></li><li><p><code>8</code> Meetings</p></li><li><p><code>9</code> Docs</p></li><li><p><code>0</code> Databases</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">Because teams can enable and disable apps, treat the visible sidebar as the source of truth. If a section shows a keyboard hint, that is the shortcut for your current workspace.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Create with C</h2><p xmlns="http://www.w3.org/1999/xhtml"><code>c</code> is the main contextual create shortcut. It activates the visible create/compose button in the current section.</p><p xmlns="http://www.w3.org/1999/xhtml">Examples:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>in Docs, <code>c</code> creates a new Doc,</p></li><li><p>in Tasks, <code>c</code> creates or resumes a new task,</p></li><li><p>in Emails, <code>c</code> composes a new email,</p></li><li><p>in Meetings, <code>c</code> creates a new meeting,</p></li><li><p>in Forms, <code>c</code> creates a new form,</p></li><li><p>in Databases, <code>c</code> creates a new database, and</p></li><li><p>in Posts, Chat, Git, AI, and other sections, <code>c</code> follows the visible create action for that section.</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">This is more reliable than memorizing separate create keys for every app.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Move through lists</h2><p xmlns="http://www.w3.org/1999/xhtml">Most item lists support keyboard selection. Use the arrow keys to move the selected item up or down in the current list.</p><p xmlns="http://www.w3.org/1999/xhtml">When the selected item moves out of view, Aamu scrolls it back into view. This makes long lists easier to scan without reaching for the mouse after every item.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Select and bulk-handle items</h2><p xmlns="http://www.w3.org/1999/xhtml">In item lists, <code>s</code> or <code>.</code> can select the current item when the list supports selection. This is useful before using bulk actions.</p><p xmlns="http://www.w3.org/1999/xhtml"><code>Esc</code> can unselect items when a selection toolbar is active.</p><p xmlns="http://www.w3.org/1999/xhtml"><code>Shift</code> with arrow navigation can also help select while moving in supported lists.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Close, reopen, delete, or toggle with X</h2><p xmlns="http://www.w3.org/1999/xhtml"><code>x</code> is used as a contextual toggle/action shortcut in several item views.</p><p xmlns="http://www.w3.org/1999/xhtml">Examples include:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>closing or reopening a task,</p></li><li><p>closing or reopening a Helpdesk ticket,</p></li><li><p>acting on the currently selected item when a supported list/action is visible.</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">Because <code>x</code> is contextual, check the visible button tooltip. If the UI says <code>Close [x]</code> or <code>Close and resolve [x]</code>, that is the action that will run.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Reply, read, and section-specific keys</h2><p xmlns="http://www.w3.org/1999/xhtml">Some sections use additional shortcuts for their own workflow. For example, Emails has shortcuts around read/unread or message actions, and some comment/reply areas expose their own keys.</p><p xmlns="http://www.w3.org/1999/xhtml">These are best learned from the visible controls, because the action depends on the current section and selection.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Internal comments with Alt+C</h2><p xmlns="http://www.w3.org/1999/xhtml">Some work items have an internal comments panel. Where the UI exposes it, <code>Alt+C</code> can open or close that internal comments area.</p><p xmlns="http://www.w3.org/1999/xhtml">This is especially useful in Helpdesk and database-style workflows where internal discussion should stay separate from customer-facing or public content.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Preview with Alt+W</h2><p xmlns="http://www.w3.org/1999/xhtml">Some textarea/editor contexts use <code>Alt+W</code> for preview. This is handled separately so it can work in writing contexts where normal global shortcuts are disabled.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Keyboard-first, not keyboard-only</h2><p xmlns="http://www.w3.org/1999/xhtml">Aamu's shortcuts are meant to make repeated work faster, not to hide the UI. You can always use the visible buttons and menus. The shortcut simply clicks the same visible action when it is available and unobstructed.</p><p xmlns="http://www.w3.org/1999/xhtml">That design matters. It keeps shortcuts discoverable, contextual, and safer than a global command system that fires regardless of what is on screen.</p><h2 xmlns="http://www.w3.org/1999/xhtml">A practical way to learn them</h2><ol xmlns="http://www.w3.org/1999/xhtml"><li><p>Use number keys for the sections you visit most.</p></li><li><p>Use <code>c</code> to create in the current section.</p></li><li><p>Use arrow keys to move through lists.</p></li><li><p>Use <code>s</code> or <code>.</code> when selecting list items.</p></li><li><p>Use <code>x</code> only when the visible action says what it will do.</p></li><li><p>Watch button tooltips for the shortcut hints.</p></li></ol><p xmlns="http://www.w3.org/1999/xhtml">After a little practice, the app starts to feel much faster because the same few keys work across many parts of the workspace.</p>

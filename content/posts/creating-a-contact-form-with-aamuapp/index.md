<p xmlns="http://www.w3.org/1999/xhtml">There are (at least) three ways to create a contact form with Aamu.app. Let’s look at the first one in detail.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Ingredients</h2><p xmlns="http://www.w3.org/1999/xhtml">The ingredients for this contact form are:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>HTML code for the form itself, which you can grab from <a target="_blank" rel="noopener noreferrer nofollow" href="https://github.com/AamuApp/contact-form">https://github.com/AamuApp/contact-form</a></p></li><li><p>Aamu.app database, with <em>forms</em> enabled</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">What you would do at your website is that you would add the HTML code, with CSS, to the appropriate location of your site, and tie the form to Aamu.app database with the Aamu.app database’s <code>endpoint url</code> and the Aamu.app database’s API key. Let’s see how to do all this.</p><h2 xmlns="http://www.w3.org/1999/xhtml">Creating the Aamu.app database</h2><p xmlns="http://www.w3.org/1999/xhtml">We have a ready-made template of the database that you can use for storing the contact form’s data — it’s under the “arrow button” just next to the “plus button” that you would normally create an empty database with. So click the arrow button and select the <em>Contact Form</em>:</p><img xmlns="http://www.w3.org/1999/xhtml" src="7a2be3fbe016098a_image.png" style="width: auto; --upload-progress: 0%;" id="a1028694-4284-4322-84e0-802f6a75db14"><h2 xmlns="http://www.w3.org/1999/xhtml">Setting up the database to accept form submissions</h2><p xmlns="http://www.w3.org/1999/xhtml">In the database settings, you should first enable the forms API:</p><img xmlns="http://www.w3.org/1999/xhtml" src="28a4b212613a2718_image.png" style="width: auto; --upload-progress: 0%;" id="48f331e6-d9b4-4ca2-a8a2-6537091c43b4"><p xmlns="http://www.w3.org/1999/xhtml">You should select the table, where the form submissions will go to:</p><img xmlns="http://www.w3.org/1999/xhtml" src="77d92ef354241682_image.png" style="width: auto; --upload-progress: 0%;" id="a02c541b-d181-41b2-a9fe-b3b29e3ef4aa"><p xmlns="http://www.w3.org/1999/xhtml">Then copy the <strong><em>Forms endpoint</em></strong>, and put it into the contact form’s HTML code, into <code>&lt;form action=”ENDPOINT HERE”&gt;</code>.</p><img xmlns="http://www.w3.org/1999/xhtml" src="f347c699fd2bb415_image.png" style="width: auto; --upload-progress: 0%;" id="e39ceeea-314e-4aa1-af82-146979e38e84"><h2 xmlns="http://www.w3.org/1999/xhtml">Creating the Form</h2><p xmlns="http://www.w3.org/1999/xhtml">To create the form, let’s start by grabbing the example HTML code from GitHub. The important part of it is the <code>&lt;form&gt;...&lt;/form&gt;</code>:</p><pre xmlns="http://www.w3.org/1999/xhtml"><code class="language-html">&lt;form id="contactForm" action="ENDPOINT_HERE" method="POST" enctype="multipart/form-data" &gt;
	&lt;input type="hidden" name="redirect-error" value="https://api.aamu.app/api/v1/forms/error"&gt;
	&lt;input type="hidden" name="redirect-success" value="https://api.aamu.app/api/v1/forms/thank-you"&gt;
	&lt;div&gt;
		&lt;input placeholder="Name" type="text" name="name" id="form_name" required autofocus&gt;
	&lt;/div&gt;
	&lt;div&gt;
		&lt;input placeholder="Email" type="text" name="email" id="form_email" required&gt;
	&lt;/div&gt;
	&lt;div&gt;
		&lt;textarea placeholder="Message" name="message" id="form_message" required&gt;&lt;/textarea&gt;
	&lt;/div&gt;
	&lt;button id="submit" type="submit"&gt;Send&lt;/button&gt;
&lt;/form&gt;</code></pre><p xmlns="http://www.w3.org/1999/xhtml">The way this form works, is first by the <code>action="ENDPOINT_HERE"</code> which sends the form the API endpoint. </p><p xmlns="http://www.w3.org/1999/xhtml">Note the two hidden fields: <code>redirect-error</code> and <code>redirect-success</code>. You can set the redirect URLs with these. When sending the form with JavaScript, you won’t need these, but they are good to be there, since JavaScript isn’t always enabled by the user.</p><h3 xmlns="http://www.w3.org/1999/xhtml">Form input field bindings</h3><p xmlns="http://www.w3.org/1999/xhtml">Another important point is how the form input fields’ names are matched to the database fields. The way the input field names are created is pretty straightforward — they are basically lowercase words with spaces turned into underscores. The correct input field names can also be seen in the <strong><em>Database settings / Forms</em></strong>. You can copy the <code>&lt;form&gt;</code>‘s HTML code there and use it in your applications.</p><p xmlns="http://www.w3.org/1999/xhtml">The form which you can get from <a target="_blank" rel="noopener noreferrer nofollow" href="https://github.com/AamuApp/contact-form" id="0ec9cb15-f76c-45a5-982b-b3d57ca1801b">GitHub</a> has some other niceties — for example it is submitted by JavaScript on the fly, without causing a round-trip at the server (Aamu.app’s server).</p><p xmlns="http://www.w3.org/1999/xhtml">For example, in our contact form database, there are three fields: <code>Name</code>, <code>Email</code>, <code>Message</code>. These correspond to form input fields <code>name</code>, <code>email</code>, <code>message</code>. </p><h3 xmlns="http://www.w3.org/1999/xhtml">That’s about it!</h3><p xmlns="http://www.w3.org/1999/xhtml">Now, if you have the form in place, point your browser to it, and test if. All the form submissions should end up in the database.</p><img xmlns="http://www.w3.org/1999/xhtml" src="b17b16742e0c429c_image.png" style="width: auto; --upload-progress: 0%;" id="e7fc64c7-5a3c-4780-a575-776e32216a5c"><p xmlns="http://www.w3.org/1999/xhtml"></p><p xmlns="http://www.w3.org/1999/xhtml"></p>

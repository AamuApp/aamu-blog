<p xmlns="http://www.w3.org/1999/xhtml">How to create a blog with Aamu.app? Here is how I have done it for Aamu.app’s official blog at <a target="_blank" rel="noopener noreferrer nofollow" href="https://aamu.app/blog/">https://aamu.app/blog/</a>.</p><p xmlns="http://www.w3.org/1999/xhtml">The ingredients of this blog are:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>A database at Aamu.app</p></li><li><p>For writing the blog posts: documents at Aamu.app</p></li><li><p>For creating the actual blog: <a target="_blank" rel="noopener noreferrer nofollow" href="https://gohugo.io/" id="3cc0064a-8cad-48d6-93df-8f6ff1fd33b7">Hugo</a> static site generator</p></li><li><p>A server (with a shell access) to host the blog</p></li><li><p>Git to publish the blog to the server (pushing the repo)</p></li><li><p>Nginx for serving the static site (git repo)</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">You can also use some other method to host the blog, for example Netlify. With this method you don’t need a server nor git. </p><p xmlns="http://www.w3.org/1999/xhtml">You can see the source code for the blog at <a target="_blank" rel="noopener noreferrer nofollow" href="https://github.com/AamuApp/aamu-blog">https://github.com/AamuApp/aamu-blog</a>.</p><h2 xmlns="http://www.w3.org/1999/xhtml">The recipe</h2><p xmlns="http://www.w3.org/1999/xhtml">Here’s how it goes, from start to finish.</p><h3 xmlns="http://www.w3.org/1999/xhtml">Create a database</h3><p xmlns="http://www.w3.org/1999/xhtml">Create a database at Aamu.app with the provided template <code>aamu-blog</code>:</p><img xmlns="http://www.w3.org/1999/xhtml" src="0ed13d54ae787f1f_image.png" style="width: auto; --upload-progress: 0%;" id="6a02906c-18cf-41b5-ab6e-2fce80636e12"><p xmlns="http://www.w3.org/1999/xhtml">You would use this database as what is known as a “Headless CMS” — the data from the database can be retrieved from the Aamu.app’s GraphQL database API.</p><p xmlns="http://www.w3.org/1999/xhtml">Inside the database you will see two tables: <code>Blog post</code> and <code>Person</code>. If you are not happy with the names, you can rename them. The table <code>Blog post</code> is for posts (<span data-node="emoji" data-emoji="🧐" data-annotation="face with monocle" style="user-select: text; font-family: &quot;Twemoji Mozilla&quot;, &quot;Apple Color Emoji&quot;, &quot;Segoe UI Emoji&quot;, &quot;Segoe UI Symbol&quot;, &quot;Noto Color Emoji&quot;, &quot;EmojiOne Color&quot;, &quot;Android Emoji&quot;, sans-serif;"><img alt="face with monocle" style="display: none; width: 1em; height: 1em;" ><span role="img" aria-label="face with monocle" style="display: inline-block;">🧐</span></span>) and <code>Person</code> is for post authors. </p><p xmlns="http://www.w3.org/1999/xhtml">You should first create a row in <code>Person</code> which you can then link into your posts:</p><img xmlns="http://www.w3.org/1999/xhtml" src="2f2cdb3a11fe1faf_image.png" style="width: auto; --upload-progress: 0%;" id="5917a2b7-ec52-427e-8736-30eec3f0f1cb"><p xmlns="http://www.w3.org/1999/xhtml">Then create a row into <code>Blog post</code> table and fill the data accordingly. <code>Body</code> field will contain the blog posts HTML code. You can write the blog posts with Docs feature at Aamu.app, which we will get to next.</p><p xmlns="http://www.w3.org/1999/xhtml">Here’s a row for a blog post. “Slug” is the url component of the post.</p><img xmlns="http://www.w3.org/1999/xhtml" src="7b2746873018dcf1_image.png" style="width: auto; --upload-progress: 0%;" id="060b0e45-1182-4550-b4b6-5e61e7e5a840"><p xmlns="http://www.w3.org/1999/xhtml">When you want to start writing the post, click the <code>Body</code> field of the row. You can write (copy-paste) HTML directly, or attach a Doc. Let’s attach a Doc:</p><img xmlns="http://www.w3.org/1999/xhtml" src="51da39447ffe1b59_image.png" style="width: auto; --upload-progress: 0%;" id="c661eb18-e621-42ce-8ff2-bdf677690af3"><h3 xmlns="http://www.w3.org/1999/xhtml">Setting up the API key</h3><p xmlns="http://www.w3.org/1999/xhtml">In order to use the database as a headless CMS, it needs to be accessed from outside Aamu.app with GraphQL. You will need to create an API key and set it up for the blog build script.</p><p xmlns="http://www.w3.org/1999/xhtml">To create an API key, go to database settings / API, and you will find what you need. Click “Generate new API key”, copy it and paste to aamu-blog’s <code>.env</code> file, which aamu-blog uses for environment variables.</p><img xmlns="http://www.w3.org/1999/xhtml" src="54bf35a44f33c97d_image.png" style="width: auto; --upload-progress: 0%;" id="d8f43f8d-896e-4ef5-93c7-43b14d721c8a"><h3 xmlns="http://www.w3.org/1999/xhtml">Writing the blog post</h3><p xmlns="http://www.w3.org/1999/xhtml">We will use the Aamu.app’s Docs feature to write the blog post (just as I’m doing now). </p><p xmlns="http://www.w3.org/1999/xhtml">Writing a doc will automatically save the HTML code to the database’s <code>Body</code> field. So, writing the blog post is extremely easy.</p><p xmlns="http://www.w3.org/1999/xhtml">You can also copy-paste the HTML to the <code>Body</code> field, if you want to write it with some other method.</p><p xmlns="http://www.w3.org/1999/xhtml">When you are ready to publish the blog post, set the publishing date, author, possibly “Hero” image and any tags you would want, and set the status to “Published”.</p><img xmlns="http://www.w3.org/1999/xhtml" src="4cf27055862b78ff_image.png" style="width: auto; --upload-progress: 0%;" id="897251ed-bc94-4e2f-a80a-d6e86e1cda40"><h3 xmlns="http://www.w3.org/1999/xhtml">Creating the blog with Hugo</h3><p xmlns="http://www.w3.org/1999/xhtml">Let’s create the actual blog now. </p><p xmlns="http://www.w3.org/1999/xhtml">The idea goes like this:</p><ul xmlns="http://www.w3.org/1999/xhtml"><li><p>The build script retrieves blog posts from Aamu.app through the GraphQL API. The script knows the timestamp when it did this the last time (the timestamp is saved into a file) and only gets the updated/new blog posts.</p></li><li><p>The same script saves them into content files, as well as any images.</p></li><li><p>Hugo is run to convert the content files into a blog</p></li></ul><p xmlns="http://www.w3.org/1999/xhtml">The source code, which we use for our blog, is at <a target="_blank" rel="noopener noreferrer nofollow" href="https://github.com/AamuApp/aamu-blog">https://github.com/AamuApp/aamu-blog</a>. You can use it as a starting point for your own blog, or just get ideas from it.</p><p xmlns="http://www.w3.org/1999/xhtml">You would typically use a development machine to create the blog and then upload (possibly with git) into the hosting server. </p><p xmlns="http://www.w3.org/1999/xhtml">You can also do all of this at a service like Netlify. If you want to use a service, they can usually be triggered to build the blog with a <em>build hook</em>. You can create such a build hook at your Aamu.app database’s settings: </p><img xmlns="http://www.w3.org/1999/xhtml" src="ce9ed1db81472d5f_image.png" style="width: auto; --upload-progress: 0%;" id="41324f84-1d74-4408-bd69-35ddd2b9a86b"><h3 xmlns="http://www.w3.org/1999/xhtml">Publishing the blog</h3><p xmlns="http://www.w3.org/1999/xhtml">Still one thing to do: to actually publish the blog at the hosting server.</p><p xmlns="http://www.w3.org/1999/xhtml">At the simplest, this can be done by uploading what Hugo created to the server. I have set up a git bare repo, with a <em>post-receive hook</em>, which handles the uploading part. Using git with a “bare repo” is a handy way to do this, but a bit more complex.</p><h3 xmlns="http://www.w3.org/1999/xhtml">That’s It!</h3><p xmlns="http://www.w3.org/1999/xhtml">Let us know if there was something that left you wondering - how could I improve this tutorial?
